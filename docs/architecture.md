# Architecture

The project is split into frontend and backend.

## Frontend

- `screens/` contains full app screens.
- `components/` contains reusable UI.
- `hooks/` contains reusable state logic.
- `services/api/` contains API calls.
- `types/` contains TypeScript types.
- `constants/` contains config and theme values.

## Backend

- `routes/` contains API endpoints.
- `services/` contains business logic.
- `repositories/` contains database queries.
- `schemas/` contains request and response models.
- `models/` contains database models.
